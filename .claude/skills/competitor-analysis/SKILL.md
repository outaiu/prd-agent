# 競合分析 スキル

競合プロダクトの体系的な分析フレームワーク。
Knowledge の既存競合データを起点に、Web調査で最新情報を補完する。

## 分析フレームワーク

### 1. 競合プロダクト特定

#### 分類
- **直接競合:** 同じ市場・同じ顧客セグメントを対象とするプロダクト
- **間接競合:** 異なるアプローチで同じ顧客課題を解決するプロダクト
- **代替手段:** プロダクトを使わずに課題を解決する方法（内製、手動作業など）

#### 特定手法
1. Knowledge の `competitors` セクションを確認
2. 対象機能に絞った競合を追加特定
3. Web検索で新興プレイヤーを調査

### 2. 機能比較マトリクス

対象機能に関連する要素を横軸、競合プロダクトを縦軸に配置:

```markdown
| 機能要素 | 自社 | 競合A | 競合B | 競合C |
|----------|------|-------|-------|-------|
| 機能1 | ◯/△/✕ | ◯/△/✕ | ◯/△/✕ | ◯/△/✕ |
| 機能2 | ◯/△/✕ | ◯/△/✕ | ◯/△/✕ | ◯/△/✕ |
| UX/UI | 評価 | 評価 | 評価 | 評価 |
| 価格帯 | ... | ... | ... | ... |
```

記号: ◯=実装済み・優れている △=部分的・改善の余地あり ✕=未実装

### 3. SWOT分析（各競合ごと）

各競合プロダクトに対して実施:

```markdown
## 競合X SWOT分析

### Strengths（強み）
- ...

### Weaknesses（弱み）
- ...

### Opportunities（機会）
- ...

### Threats（脅威）
- ...
```

### 4. ポジショニングマップ

2軸で競合をマッピング:
- 推奨軸の例:
  - 横軸: 専門特化 ←→ 汎用
  - 縦軸: 個人向け ←→ 法人向け
  - 横軸: 低価格 ←→ 高価格
  - 縦軸: セルフサービス ←→ フルサービス

### 5. 差別化ポイント特定

以下の観点で自社の差別化機会を整理:

1. **機能差別化:** 競合にない機能、または大幅に優れている機能
2. **体験差別化:** UX/UI、オンボーディング、サポートの質
3. **価格差別化:** 価格モデル、コストパフォーマンス
4. **ネットワーク差別化:** ユーザーベース、マーケットプレイスの厚み
5. **データ差別化:** 蓄積データによる優位性（レコメンド精度等）

## 調査手法

### Web検索クエリ例
- `"{競合名}" {機能名} 機能 リリース`
- `"{競合名}" vs "{競合名}" 比較`
- `{業界名} {機能カテゴリ} トレンド {年}`
- `"{競合名}" pricing 料金`

### 情報ソース優先度
1. 競合の公式サイト・ヘルプセンター
2. プレスリリース・ブログ
3. 業界メディア・比較サイト
4. ユーザーレビュー（G2, ITreview等）
5. IR資料・決算説明資料

## 出力フォーマット

### 分析サマリー（PRDへの入力用）

```yaml
competitor_analysis:
  target_feature: "{対象機能名}"
  analyzed_at: "{YYYY-MM-DD}"
  competitors:
    - name: "{競合名}"
      type: "direct|indirect"
      feature_status: "{対象機能の実装状況}"
      strengths: [...]
      weaknesses: [...]
      key_differentiator: "{主な差別化要素}"
  differentiation_opportunities:
    - "{差別化機会1}"
    - "{差別化機会2}"
  market_gaps:
    - "{市場のギャップ}"

    - "{脅威}"
```

## 分析ステップ詳細（pm-skills competitor-analysis より）

### 6段階分析プロセス

1. **Market Scoping（市場定義）**
   - 市場・業界・対象顧客ベースを定義
   - 市場規模と成長トレンドを把握
   - 主要顧客セグメントとユースケースを特定
   - 市場の主要成功要因を明確化

2. **Competitor Identification（競合特定）**
   - 5社の直接競合をWeb検索で特定
   - リーダー、チャレンジャー、ニッチプレイヤーのポジション整理
   - 推定市場シェアまたはポジショニング
   - 隣接・間接競合の把握

3. **Competitive Intelligence（競合情報収集）**
   - 各競合のポジショニング、機能、価格、GTM戦略を調査
   - 公式サイト、価格ページ、顧客レビューを重点的に調査
   - プロダクトローンチ、資金調達、経営陣の動向をWeb検索

4. **Strengths & Weaknesses（強み弱み分析）**
   - 競合の能力、制限、市場ポジショニングを評価
   - 各競合の一意の競争優位性を特定
   - 顧客満足度・リテンションのシグナルを確認

5. **Differentiation Mapping（差別化マッピング）**
   - ギャップ、重複、差別化機会を特定
   - 競合がカバーしていない顧客ニーズを発見
   - JTBDの観点から効果的に解決されていない課題を特定

6. **Strategic Synthesis（戦略的統合）**
   - 競争ダイナミクスと将来の脅威についてのインサイト
   - 推奨ポジショニングと強調すべき差別化要素
   - 12〜18ヶ月の競争リスクと機会

## ベストプラクティス

- 現在の競合サイト、価格ページ、顧客レビューを調査する
- 直接競合と隣接する代替手段を区別する
- 複数ソースで競合インサイトを検証する
- 明白な差別化機会と微妙な差別化機会の両方を特定する
- 市場シェアを獲得しつつある新興競合に注意する
- 長期的な競争ダイナミクスと市場シフトを考慮する
